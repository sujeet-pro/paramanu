## What does Paramanu mean?

**Paramanu** (परमाणु) is a Sanskrit term meaning **atom** — the smallest, indivisible particle of matter. In the ancient Indian Vaisheshika school of philosophy, paramanu represented the fundamental building blocks from which all physical reality is composed.

This design system carries the same idea: **the smallest, composable building blocks for crafting web interfaces**. Each token, style, and component is a paramanu — minimal, purposeful, and designed to combine into something greater.

## Why Paramanu?

Built for teams that care about performance, accessibility, and developer experience.

</div>

<div class="pm-features">
  <div class="pm-feature">
    <div class="pm-feature-icon">&#x1f3a8;</div>
    <h3>CSS-First, Zero Runtime</h3>
    <p>Built entirely on CSS custom properties and <code>@layer</code> cascade ordering. No JavaScript needed for styling — your buttons, inputs, and layouts work without a framework.</p>
  </div>
  <div class="pm-feature">
    <div class="pm-feature-icon">&#x1f9e9;</div>
    <h3>Three Consumption Modes</h3>
    <p>Drop in via <strong>CDN</strong> with human-readable classes, use as <strong>CSS Modules</strong> for scoped styles, or import <strong>React components</strong> with full type safety.</p>
  </div>
  <div class="pm-feature">
    <div class="pm-feature-icon">&#x2665;</div>
    <h3>Accessible by Default</h3>
    <p>WCAG 2.2 AA compliant. Focus-visible indicators, semantic HTML, ARIA attributes, and automated axe-core testing baked in.</p>
  </div>
  <div class="pm-feature">
    <div class="pm-feature-icon">&#x2728;</div>
    <h3>Modern CSS</h3>
    <p>CSS nesting, <code>light-dark()</code> for automatic dark mode, <code>@layer</code> for cascade control, container queries, and <code>@scope</code>.</p>
  </div>
  <div class="pm-feature">
    <div class="pm-feature-icon">&#x1f3ad;</div>
    <h3>5 Built-in Themes</h3>
    <p>Material, Ant Design, Bootstrap, Dark Modern, and Light Modern. Swap a class name to re-skin your entire app instantly.</p>
  </div>
  <div class="pm-feature">
    <div class="pm-feature-icon">&#x1f4e6;</div>
    <h3>Tree-Shakeable</h3>
    <p>Import only what you use. Each component group is a separate package — no monolithic bundles, no unused CSS.</p>
  </div>

## Three Ways to Use

</div>

<div class="pm-consumption-tabs">
<Tabs syncKey="consumption-mode">
  <TabItem label="CDN">
    Link the stylesheet and start using classes immediately. No build tools needed.

    ```html
    <link rel="stylesheet" href="https://unpkg.com/@paramanu/cdn/dist/paramanu.css">

    <button class="pm-btn pm-btn--primary pm-btn--md">
      Get Started
    </button>
    ```

  </TabItem>
  <TabItem label="CSS Modules">
    Import as CSS Modules for scoped class names in bundled applications.

    ```js
    import classes from "@paramanu/buttons-js/modules/button/map"

    const btn = document.createElement("button")
    btn.className = classes["pm-btn"] + " " + classes["pm-btn--primary"]
    btn.textContent = "Get Started"
    ```

  </TabItem>
  <TabItem label="React">
    Import typed React components with full prop autocompletion.

    ```tsx
    import { Btn } from "@paramanu/buttons-react"

    export default function App() {
      return <Btn variant="primary" size="md">Get Started</Btn>
    }
    ```

  </TabItem>
</Tabs>

## Design Principles

</div>

<div class="pm-principles">
  <div class="pm-principle">
    <h3>Minimal by Design</h3>
    <p>Every component does one thing well. No bloated bundles, no hidden dependencies, no surprise side effects.</p>
  </div>
  <div class="pm-principle">
    <h3>Composable</h3>
    <p>Tokens feed into components, components combine into patterns. Override any layer without fighting specificity wars.</p>
  </div>
  <div class="pm-principle">
    <h3>Progressive Enhancement</h3>
    <p>Starts with semantic HTML and CSS. Layer on JavaScript interactivity only when needed. Works without JS, shines with it.</p>
  </div>
  <div class="pm-principle">
    <h3>Themeable</h3>
    <p>Swap design tokens at the root level to re-skin everything. Built-in light-dark() support means dark mode with one CSS property.</p>
  </div>
</div>
