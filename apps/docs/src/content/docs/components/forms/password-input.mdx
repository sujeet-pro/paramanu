The Password Input wraps a standard input with a visibility toggle, allowing users to reveal or conceal their password.

## Basic Usage

<ComponentPreview
  htmlCode={`<div class="pm-password-input pm-password-input--outline pm-password-input--md">\n  <input class="pm-input pm-input--outline pm-input--md" type="password" placeholder="Enter password..." />\n  <button type="button" class="pm-password-input__toggle" aria-label="Show password">Show</button>\n</div>`}
  reactCode={`import { PasswordInput } from "@paramanu/forms-react"\n\nexport default function App() {\n  return <PasswordInput placeholder="Enter password..." />\n}`}
/>

## Accessibility

| Feature | Detail |
|---|---|
| Toggle button | `aria-label` updates between "Show password" and "Hide password" |
| `aria-invalid` | Applied when `invalid` is true |
| Keyboard | Toggle button is focusable via Tab |

## Best Practices

- Always pair with a label for the password field.
- Use the visibility toggle to help users verify their input.

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `variant` | `"outline" \| "filled" \| "unstyled"` | `"outline"` | Visual variant |
| `size` | `"xs" \| "sm" \| "md" \| "lg"` | `"md"` | Input size |
| `invalid` | `boolean` | `false` | Invalid state |
| `disabled` | `boolean` | `false` | Disabled state |
| `fullWidth` | `boolean` | `false` | Full-width layout |

## CSS Classes

| Class | Description |
|---|---|
| `pm-password-input` | Wrapper base class |
| `pm-password-input__toggle` | Toggle button |
| `pm-password-input--visible` | Visible state |
