## Accessibility

- Uses semantic `<button>` element by default
- Focus-visible ring: 2px solid with 2px offset (WCAG 2.2 AA compliant)
- Disabled buttons use both `aria-disabled="true"` and the `disabled` attribute
- Supports `aria-label` for icon-only buttons
- Color contrast meets WCAG AA requirements for all variants

## Best Practices

- **Use the right variant** for the action's importance: `primary` for the main action, `secondary` for alternatives, `danger` for destructive actions, and `ghost` for low-emphasis actions.
- **One primary button per section** — avoid multiple primary buttons competing for attention.
- **Use clear, concise labels** — prefer action verbs like "Save", "Delete", "Submit" over vague labels like "OK" or "Click here".
- **Indicate loading states** for async actions to prevent double-clicks.
- **Always include a `type` attribute** in forms — use `type="submit"` for form submission and `type="button"` for non-submit actions.

## Do's and Don'ts

| Do | Don't |
|---|---|
| Use `variant="danger"` for destructive actions | Use `variant="primary"` styled red with custom CSS |
| Disable buttons when action is unavailable | Hide buttons and confuse users |
| Use `fullWidth` for mobile or single-action layouts | Use `fullWidth` for every button on the page |
| Provide `aria-label` for icon-only buttons | Leave icon-only buttons without accessible names |

## React Props

The `Button` component extends all native `<button>` HTML attributes.

| Prop | Type | Default | Description |
|---|---|---|---|
| `variant` | `"primary" \| "secondary" \| "danger" \| "ghost"` | `"primary"` | Visual style of the button |
| `size` | `"sm" \| "md" \| "lg"` | `"md"` | Button size |
| `disabled` | `boolean` | `false` | Disabled state — sets both `disabled` and `aria-disabled` |
| `fullWidth` | `boolean` | `false` | Makes the button span full container width |
| `type` | `"button" \| "submit" \| "reset"` | `"button"` | HTML button type |
| `className` | `string` | — | Additional CSS class names |
| `children` | `ReactNode` | — | Button content |
| `ref` | `Ref<HTMLButtonElement>` | — | Forwarded ref to the underlying `<button>` |

## Theming

The button uses `light-dark()` for automatic dark mode support. Set `color-scheme: dark` on a parent element to switch to dark mode colors.

```css
:root { color-scheme: light dark; }
```

### CSS Custom Properties

The button component reads design tokens from `@paramanu/tokens`. Override these at the `:root` level to customize the theme globally:

```css
:root {
  --pm-color-primary: #2563eb;
  --pm-color-danger: #dc2626;
}
```
