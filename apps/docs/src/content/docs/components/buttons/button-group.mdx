Groups related buttons together with consistent spacing, optional attached borders, and horizontal or vertical layout.

## Basic Usage

<ComponentPreview
  htmlCode={`<div class="pm-button-group pm-button-group--horizontal" role="group" aria-label="Actions">
  <button class="pm-button pm-button--secondary pm-button--md">Left</button>
  <button class="pm-button pm-button--secondary pm-button--md">Center</button>
  <button class="pm-button pm-button--secondary pm-button--md">Right</button>
</div>`}
  reactCode={`import { ButtonGroup, Button } from "@paramanu/buttons-react"

export default function App() {
  return (
    <ButtonGroup aria-label="Actions">
      <Button variant="secondary">Left</Button>
      <Button variant="secondary">Center</Button>
      <Button variant="secondary">Right</Button>
    </ButtonGroup>
  )
}`}
/>

## Accessibility

- Uses `role="group"` with `aria-label` for screen reader context
- Keyboard navigation follows standard tab order through buttons

## Best Practices

- Always provide an `aria-label` describing the group's purpose
- Use consistent button variants within a group
- Limit groups to 2-5 buttons for clarity

</TabItem>

## React Props

Extends all native `<div>` HTML attributes.

| Prop | Type | Default | Description |
|---|---|---|---|
| `orientation` | `"horizontal" \| "vertical"` | `"horizontal"` | Layout direction |
| `attached` | `boolean` | `false` | Merge button borders |
| `fullWidth` | `boolean` | `false` | Stretch to container width |
| `children` | `ReactNode` | -- | Button elements |

## CSS Classes

| Class | Purpose |
|---|---|
| `pm-button-group` | Base styles |
| `pm-button-group--horizontal` | Horizontal layout |
| `pm-button-group--vertical` | Vertical layout |
| `pm-button-group--attached` | Merged borders |
| `pm-button-group--full-width` | Full width |
